# EVOSCZ — Evolutionary architecture of schizophrenia common-variant heritability

Analysis code accompanying the manuscript *Common schizophrenia heritability concentrates in an evolutionarily young, brain-regulatory subset of fine-mapped credible-set variants* (Çiçek, Altunç & Demirel, 2026).

We partition fine-mapped PGC3 schizophrenia credible-set variants by coalescent allele age, ancient-DNA/haplotype selection signals, and brain-versus-blood regulatory specificity (3-component Gaussian mixture), and show that the youngest, brain-regulatory subset concentrates common-variant heritability (stratified LD-score regression, baseline-LD v2.2). All subsets are deeply ancient (cluster medians ≈113–508 kyr, pre-out-of-Africa). Allele ages are GEVA/Atlas-of-Variant-Age estimates in **generations**, converted to years at 28.1 yr generation⁻¹.

## Repository layout
- `scripts/phase11/` — variant master build (GEVA ages, GTEx v10 brain/blood cis-eQTL → brain_spec, gnomAD constraint)
- `scripts/phase13/` — within-locus partial-rank correlations + bootstrap
- `scripts/phase14b/` — 3-component Gaussian mixture clustering (the three age-ordered subsets)
- `scripts/phase14e/`, `scripts/phase18/` — partitioned LD-score regression (baseline-LD v2.2)
- `scripts/phase14h/j/f/p/` — East-Asian / sex-stratified / Alzheimer / Atlas-age & AFR analyses
- `scripts/phase17/` — robustness (MAPT, power-matched, k-sensitivity, permutation null)
- `scripts/phase19/` — polarity / cross-ancestry replication
- `scripts/figures/` — the five main-figure scripts (read the result files, write the published panels)
- `scripts/phase14e/ldsc/` — vendored LDSC v1.0.1 with a Python 2→3 port (algorithm unchanged)

## Environment variables (paths are configurable; no personal paths in code)
| Variable | Points to |
|---|---|
| `EVOSCZ_ROOT` | repository root (default: inferred from the script location) |
| `EVOSCZ_SCRATCH` | working/scratch directory for large intermediate files (default: `$EVOSCZ_ROOT/scratch`) |
| `GTEX_V10_DIR` | GTEx v10 cis-eQTL data directory |
| `CELLTYPE_PEAKS_DIR` | single-cell / cell-type peak data |
| `PGC_SUMSTATS_DIR` | PGC3 SCZ summary statistics (EUR / EAS / AFR / sex strata) |
| `AD_SUMSTATS_DIR` | Wightman 2021 Alzheimer's summary statistics |
| `ATLAS_DIR` | Atlas of Variant Age (GEVA) downloads |
| `CONDA_BASE` | conda/miniforge install prefix (used by the SLURM scripts only) |

### Setup
```bash
export EVOSCZ_ROOT=/path/to/EVOSCZ          # repository root
export PGC_SUMSTATS_DIR=/path/to/pgc        # set the data dirs you need:
export GTEX_V10_DIR=/path/to/gtex_v10       #   GTEX_V10_DIR, ATLAS_DIR,
export ATLAS_DIR=/path/to/atlas_variant_age #   AD_SUMSTATS_DIR, CELLTYPE_PEAKS_DIR
```
Python scripts fall back to inferring `EVOSCZ_ROOT` from the script location, so they run without setup. The shell (`.sh`) scripts have no fallback — `EVOSCZ_ROOT` (and the relevant data dirs) must be **exported** before running them.

**SLURM (`.slurm`) scripts** are HPC templates: export `EVOSCZ_ROOT`, `EVOSCZ_SCRATCH` and `CONDA_BASE`, and adjust the `#SBATCH` `--partition` / `--account` / `--time` / `--cpus-per-task` directives and the relative `logs/` paths for your own cluster before submitting (`mkdir -p logs` first).


## Reproduction order
`phase11` (variant master) → `phase13` (within-locus) → `phase14b` (GMM clusters) → `phase14e`/`phase18` (S-LDSC) → `phase14f/h/j/p` (AD / EAS / sex / Atlas-AFR) → `phase17` (robustness) → `phase19` (replication) → figures (`scripts/figures/`).

## Figures → scripts → data
Each main figure is regenerated by a script in `scripts/figures/` that reads the verified result files (figures are written to `$EVOSCZ_FIG_OUT`, default `$EVOSCZ_ROOT/figures`):

| Figure | Script | Reads |
|---|---|---|
| Fig 1 — GMM cluster projections | `scripts/figures/fig1_gmm_corner.py` | `results/phase14b/P14b_v3_cluster_assignments.tsv.gz` |
| Fig 2 — partitioned heritability | `scripts/figures/fig2_partitioned_heritability.py` | `results/phase18_v22/*.results` |
| Fig 3 — cell-type eQTL coverage | `scripts/figures/fig3_celltype_eqtl_coverage.py` | `results/phase14d/P14d_*.tsv` |
| Fig 4 — within-locus partial-rank | `scripts/figures/fig4_within_locus_partial_rank.py` | `results/phase13/*` |
| Fig 5 — African-lineage deep time | `scripts/figures/fig5_afr_deeptime.py` | `results/phase14p6/P14p6b_AFR_*.{parquet,tsv}` |

Main-text statistics map to the same result files: cluster sizes/ages → `results/phase14b/`; partitioned-heritability enrichments/Z (Table with C0/C1/C2) → `results/phase18_v22/` (`*.results` + `V2.2_ALL_VALUES.json`); within-locus correlations → `results/phase13/P13b*,P13c*`.
